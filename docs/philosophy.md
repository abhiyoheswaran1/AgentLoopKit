# Philosophy

AgentLoopKit starts from one belief: code is not the only artifact that matters.

When a coding agent changes a repository, the reviewer needs scope, constraints, verification evidence, and rollback notes. Without those, even working code can be expensive to trust.

AgentLoopKit keeps the method inside the repo so any compatible agent can read it before editing. The tool does not need a model, server, account, or database.

The loop is simple:

- specify the work
- constrain the risk
- plan the diff
- implement narrowly
- verify honestly
- review the result
- hand off evidence
