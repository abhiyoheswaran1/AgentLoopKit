# AgentLoopKit v0.28.0

AgentLoopKit 0.28.0 turns the CLI into a local acceptance layer for agent-generated code. The release focuses on review readiness: task evidence, verification proof, PR preparation, run history, and read-only context for coding agents.

## Highlights

- Added `agentloop ship`, a review-readiness flow that checks the active task, changed files, verification evidence, gates, handoff status, risk flags, and writes a local ship report.
- Added deterministic review-readiness scoring from `0` to `100`. The score explains evidence quality and review preparedness; it does not claim to measure code quality.
- Added `agentloop prepare-pr` for PR title/body generation from the active task, git status, verification reports, ship reports, gates, risk notes, and rollback notes.
- Added GitHub-comment Markdown output for `ship` and `prepare-pr`, so CI can post review context without the CLI handling GitHub tokens.
- Added `.agentloop/runs/`, `agentloop runs`, `agentloop show-run <id>`, and `agentloop intent <file>` for local run history and file-intent lookup.
- Added `agentloop maintainer-check` to help maintainers review agent-assisted PRs for task evidence, verification freshness, risky areas, generated files, and dependency changes.
- Added `agentloop review-context` for a read-only local snapshot that combines status, gates, policies, artifacts, recent runs, and latest ship evidence.
- Added read-only MCP tools for ship evidence, run details, file intent, maintainer checks, policy status, gate status, artifact inventory, and review context.

## Safety And Trust

- Hardened AgentLoopKit artifact writes and reads against symlink escapes and unsafe configured paths.
- Added repo-relative display paths in public-facing outputs for verification, handoff, ship, prepare-pr, and run-ledger flows.
- Added `--redact-paths` to public log surfaces that may otherwise expose local Git roots.
- Hardened Markdown output across reports, handoffs, release notes, CI summaries, doctor output, release checks, task output, and npm status output.
- Kept npm safety rules intact: no telemetry, no postinstall script, no API keys, no cloud backend, and no hidden network calls.

## CLI And Automation

- Added `agentloop release-check` for local release-readiness checks.
- Added `agentloop artifacts` for read-only local evidence inventories.
- Added `agentloop doctor --strict`, `agentloop status --brief`, and `agentloop verify --timeout-ms`.
- Added `agentloop verify --task-commands` and task-command metadata in JSON output.
- Added JSON output and structured errors across more CLI commands.
- Added cross-platform CLI smoke coverage for Ubuntu, macOS, and Windows.
- Improved the GitHub Action wrapper so workflows can run the CLI with safer argument handling.

## Documentation

- Refreshed the README around the review-readiness loop:
  `init` -> `create-task` -> agent work -> `ship` -> `prepare-pr`.
- Moved detailed command behavior into `docs/cli-reference.md`.
- Added an end-to-end example covering init, doctor, task creation, verification, handoff, gates, artifacts, and release readiness.
- Updated MCP, GitHub Actions, distribution, release, and safety docs around the current local-first scope.

## Install

```bash
npx agentloopkit@latest init
```

Or pin this release:

```bash
npx agentloopkit@0.28.0 init
```

## Verify

```bash
npx agentloopkit@0.28.0 version
npx agentloopkit@0.28.0 doctor
```
