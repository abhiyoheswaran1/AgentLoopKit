# Target User Personas

Internal target-user model for product decisions. This is simulated persona work, not real user research.

## Indie Hacker Using Codex

- Builds products solo.
- Uses Codex in automode.
- Wants fast shipping without losing control.
- Needs simple task contracts, verification, and PR summaries.
- Wants to install with one command.

## Claude Code Power User

- Uses Claude Code daily.
- Has custom prompts and skills.
- Wants repo-specific discipline.
- Wants better handoff and review outputs.
- May already use skill systems like Superpowers.

## Cursor Developer

- Works mostly inside IDE.
- Wants `AGENTS.md` and repo guidance Cursor agents can follow.
- Likes low-friction setup.
- Does not want a separate workflow that fights the IDE.

## Open Source Maintainer

- Reviews agent-assisted PRs.
- Wants contributors and agents to provide clear verification evidence.
- Wants less messy diffs.
- Wants a reviewer-friendly summary.

## Small Team CTO

- Wants a consistent workflow for humans and agents.
- Wants lightweight governance without buying an enterprise platform.
- Could later pay for team features.
- Wants this to work across multiple repos.

## Automation-Skeptical Senior Engineer

- Does not trust unstructured automation.
- Might accept AgentLoopKit if it improves reviewability, test discipline, and safety.
- Wants deterministic, inspectable outputs.

## Agency Developer

- Works across many client repos.
- Wants a reusable setup that can be installed into every repo.
- Wants predictable handoff summaries and verification reports.
- Wants client-friendly documentation.

## Platform Engineer

- Wants standards for agentic work across internal repos.
- Wants policy templates, review gates, and safe defaults.
- Wants to avoid each team inventing its own agent-assisted workflow.
